# dotfiles 運用原則

このリポジトリで作業するときに従う原則をまとめる。Claude Code のグローバル設定は `claude/CLAUDE.md` を参照する。ここでは本リポジトリ固有のルールのみを扱う。

## シンプル優先

- 複雑な分岐や抽象化より、読み下せる素朴な設定を選ぶ
- 不要になった設定はコメントアウトで残さず削除する
- YAGNI: 今使っていない設定を「いつか使うかも」で残さない
- プラグイン・ツールを追加する前に、既存の仕組みで解決できないかを先に検討する
- 一つの関心事は一つのファイルに閉じる（OS 判定は現行の `bash/<os>_aliases` パターンに倣う）

## シークレット管理

- 以下はコミット禁止
  - `.env`、値つきの `.envrc`
  - トークン、API キー、パスワード
  - SSH 秘密鍵、GPG 秘密鍵
  - 個人メールアドレス、実名、社内固有情報
- 秘密を含む設定は `~/.zshrc.local` / `~/.bashrc.local` / `~/.gitconfig.local` のようなローカルファイルに切り出し、本体から存在時のみ `source` する
- 例を配布する場合は `*.example` として値を伏せた雛形のみをコミットする
- コミット前に `git diff --staged` で秘密が混入していないか必ず確認する
- 誤ってコミットした場合は履歴書き換えと該当シークレットのローテーションを行う

## 冪等性・壊さない運用

- `*_link.sh` は何度実行しても同じ結果になるように保つ（既存 symlink の上書きは可、実ファイルには触れない）
- `ln -sf` で symlink を更新する場合でも、リンク先が通常ファイルであれば上書きせず警告を出す
- `Brewfile` の追加・削除では `brew bundle` 実行時に破壊的変更が起きないことを確認する。`brew bundle cleanup` はユーザー判断に委ねる
- 変更前の状態が git で追える状態でのみ操作する。未コミットの変更があるファイルは触らない
- 新しい link スクリプトは dry-run でパスと既存ファイルの状態を表示できるようにする
